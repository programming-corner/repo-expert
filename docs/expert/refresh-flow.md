# Flow: Refresh — Doc Staleness Detection

> How repo-expert detects stale documentation and regenerates it using pure git-based tracking. Triggered when the user says "refresh", "rescan", "update knowledge", or "re-learn".

## Entry Point
- **Trigger**: User invokes `/repo-expert rescan` (or synonyms)
- **Caller**: Developer in a Claude Code session
- **Auth required**: No

## Staleness Detection

Each generated doc carries frontmatter:
```yaml
---
generated: YYYY-MM-DD
git_sha: <sha_at_generation_time>
source_files:
  - path/to/relevant/file.md
---
```

Detection is purely git-based — zero API cost:
```bash
git diff <stored_git_sha>..HEAD --name-only
```
If any listed `source_files` appear in the changed file list → doc is stale.
Stale docs are marked `⚠️ stale` in KNOWLEDGE.md's flows index. They are only surfaced to the user when the stale doc is needed for the current question.

## Primary Refresh Mechanism: doc-check.sh

`doc-check.sh` is a plain bash script developers run **instead of** `git commit` when they want docs kept in sync.

**Setup** (once per repo):
```bash
cp .claude/skills/repo-expert/doc-check.sh ./doc-check.sh
chmod +x doc-check.sh
```

**Workflow:**
```bash
git add src/orders/orders.service.ts   # stage code changes
./doc-check.sh "feat: add retry logic" # run instead of git commit
```

**What it does (in order):**
1. Aborts if nothing is staged
2. Scans `docs/expert/*.md` and `KNOWLEDGE.md` for stale frontmatter
3. For each stale doc: calls `claude --print` (headless, non-interactive) to regenerate
4. Shows a coloured diff — developer chooses `y / n / e(dit)` per doc
5. Approved docs are `git add`-ed automatically
6. Prompts for commit message (or accepts it as `$1`); appends `[docs refreshed]` when docs were updated
7. Commits all staged changes (code + approved docs) in a single atomic commit
8. Optionally pushes immediately

**Configurable via env vars:**

| Variable | Default | Purpose |
|---|---|---|
| `DOCS_DIR` | `docs/expert` | Where flow docs live |
| `KNOWLEDGE_FILE` | `KNOWLEDGE.md` | Master index file |
| `SKILL_DIR` | `.claude/skills/repo-expert` | Path to this skill |

## How Claude Regenerates a Doc (invoked by doc-check.sh)

When `claude --print` is called for a stale doc:
1. Read the current doc — preserve its section structure and frontmatter keys
2. Read every file listed in `source_files` that appears in the git diff
3. Regenerate using the template from `references/doc-templates.md`
4. Update frontmatter: `git_sha` → current HEAD SHA, `generated` → today's date
5. Output **only** the complete updated doc — no explanation, no markdown fences

**Key constraint:** Never invent information. If a source file no longer contains something the doc referenced, remove that section.

## Business Rules
- Staleness detection runs in milliseconds with zero API cost
- The Claude API is only invoked when the developer explicitly approves regeneration
- Stale docs are only flagged to the user when they're needed for the current question
